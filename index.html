Core Workflow - claude.yml

모든 시나리오에서 재사용되는 핵심 워크플로우입니다.

flowchart LR subgraph Inputs I1[prompt] I2[target_number] I3[target_type] end subgraph Process P1{Valid?} P2[ai-actions] P3[Skip] end subgraph Execution X1[Parse Context] X2[Call Claude API] X3[Execute gh command] end I1 --> P1 I2 --> P1 I3 --> P1 P1 -->|Yes| P2 P1 -->|No| P3 P2 --> X1 X1 --> X2 X2 --> X3 style P2 fill:#6366f1,color:#fff style X2 fill:#8b5cf6,color:#fff 
workflow_call:
  inputs:
    prompt:
      description: 'Claude prompt'
      required: true
      type: string
    target_number:
      description: 'PR or Issue number'
      required: true
      type: number
    target_type:
      description: 'pr or issue'
      type: string
      default: 'pr'
  secrets:
    CLAUDE_CODE_OAUTH_TOKEN:
      required: true

Automation Scenarios

5가지 자동화 시나리오

SCENARIO 01
Auto Code Review

PR이 생성되면 자동으로 코드 리뷰를 수행합니다.

  • 코드 품질 분석
  • 보안 취약점 탐지
  • 성능 개선 제안
SCENARIO 02
Auto Bug Fix

bug 라벨이 붙은 이슈를 자동으로 분석합니다.

  • 버그 원인 분석
  • 수정 코드 제안
  • 테스트 케이스 추천
SCENARIO 03
Auto Documentation

새 파일이 추가되면 자동으로 문서를 생성합니다.

  • JSDoc 자동 생성
  • 사용 예제 추가
  • README 업데이트
SCENARIO 04
Test Coverage

테스트가 필요한 코드에 테스트를 제안합니다.

  • 테스트 케이스 생성
  • 엣지 케이스 식별
  • 모킹 전략 제안
SCENARIO 05
Dependency Review

의존성 업데이트 PR을 자동으로 검토합니다.

  • 보안 취약점 검사
  • Breaking Changes 분석
  • 호환성 확인

Scenario 5: Dependency Review

Sequence Diagram

sequenceDiagram autonumber participant B as Dependabot participant G as GitHub participant W as Workflow-5 participant C as claude.yml participant A as Claude API B->>G: Create deps PR G->>W: Trigger PR event W->>W: Check package.json W->>W: Verify bump title W->>C: workflow_call C->>A: Send package diff A->>A: Check CVE A->>A: Review breaking A->>A: Check license A-->>C: Return review C->>G: gh pr comment G-->>B: Security review done

  pull_request:
    types: [opened, synchronize]
    paths:
      - 'package.json'
      - 'package-lock.json'

# if: title contains 'bump' or 'deps'

Implementation Guide

핵심 기술 요소

GITHUB_TOKEN Limitation Solution

flowchart LR subgraph Problem[Problem: Cannot Trigger] P1[Event] --> P2[Workflow-1] P2 --> P3[Comment] P3 -.->|X| P4[Workflow-2] end subgraph Solution[Solution: workflow_call] S1[Event] --> S2[Wrapper] S2 --> S3[claude.yml] S3 --> S4[ai-actions] S4 --> S5[Result] end style P4 fill:#ef4444,color:#fff style S3 fill:#10b981,color:#fff style S4 fill:#10b981,color:#fff
GITHUB_TOKEN Limitation: GITHUB_TOKEN은 보안상 새로운 워크플로우를 트리거할 수 없습니다. workflow_call 패턴으로 직접 호출하여 해결합니다.

Required Configuration

# Repository Secrets
CLAUDE_CODE_OAUTH_TOKEN: "your-token"

# Workflow call with secrets
jobs:
  call-claude:
    uses: ./.github/workflows/claude.yml
    with:
      prompt: "Review this PR"
      target_number: ${{ github.event.pull_request.number }}
    secrets: inherit
