Implementation Note

Important: ai-actions는 push 이벤트를 직접 지원하지 않습니다. 따라서 이 시나리오는 push 이벤트 발생 시 @claude 멘션이 포함된 이슈를 생성하는 간접적인 방식으로 구현되었습니다.

Documentation Output

  • JSDoc 스타일 함수 문서화
  • 매개변수 및 반환값 설명
  • 사용 예제 코드

Implementation Guide

시스템 구축을 위한 핵심 기술 요소

Key Technical Solutions

GITHUB_TOKEN Limitation: GITHUB_TOKEN은 보안상 새로운 워크플로우를 트리거할 수 없습니다. 이를 해결하기 위해 workflow_call 패턴을 사용하여 직접 워크플로우를 호출합니다.
flowchart LR subgraph Problem["기존 방식 (실패)"] A1[Event] --> A2[Workflow 1] A2 -->|GITHUB_TOKEN| A3[Comment] A3 -.->|X Cannot Trigger| A4[Workflow 2] end subgraph Solution["해결 방식 (성공)"] B1[Event] --> B2[Wrapper Workflow] B2 -->|workflow_call| B3[claude.yml] B3 --> B4[ai-actions] B4 --> B5[Result] end style A4 fill:#ef4444,color:#fff style B3 fill:#10b981,color:#fff style B4 fill:#10b981,color:#fff

Required Secrets

# Repository Settings > Secrets and variables > Actions CLAUDE_CODE_OAUTH_TOKEN: "your-claude-oauth-token" # workflow_call에서 secrets 전달 jobs: call-claude: uses: ./.github/workflows/claude.yml with: prompt: "Review this PR" target_number: ${{ github.event.pull_request.number }} secrets: inherit # 중요: secrets 상속

Claude Args Configuration

# claude.yml에서 허용할 명령어 설정 claude_args: | --allowedTools "Bash(gh pr:*)" --allowedTools "Bash(gh issue:*)" --allowedTools "Bash(git diff:*)" --allowedTools "Bash(git log:*)" --allowedTools "Bash(git show:*)"
Security Note: allowedTools를 명시적으로 제한하여 Claude가 실행할 수 있는 명령어를 최소화합니다. gh와 git 명령어만 허용하여 보안을 강화합니다.
